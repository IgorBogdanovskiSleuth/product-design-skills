Here are some ideas for new skills:

## Top Priority Skills

### 1. usability-test-planner
Create comprehensive usability test plans and documentation.

**Features:**
- Generate test plans with objectives, methodology, and timeline
- Create participant screeners and recruitment criteria
- Build test scripts with task scenarios and success criteria
- Document testing protocols and moderation guides
- Track test sessions and synthesize findings
- Export to various formats (Markdown, Notion, Google Docs)

**Triggers:** "plan usability test", "create test script", "usability test"

**Use cases:**
- Planning moderated usability tests
- Creating unmoderated test protocols
- Documenting test findings and recommendations

---

### 2. user-persona-builder
Generate and maintain user personas from research data.

**Features:**
- Create structured personas with demographics, goals, pain points, behaviors
- Update personas based on new research insights
- Link personas to features, journeys, and design decisions
- Generate persona cards and presentations
- Export to Markdown, Notion, Miro-compatible formats
- Track persona evolution over time

**Triggers:** "create persona", "update persona", "build persona for [name]"

**Use cases:**
- Synthesizing user research into actionable personas
- Maintaining living personas that evolve with new data
- Creating persona documentation for team alignment

---

### 3. research-synthesis
Synthesize user research into themes, insights, and recommendations.

**Features:**
- Process interview transcripts and notes
- Extract key quotes, patterns, and pain points
- Identify themes using affinity mapping
- Generate insight statements with evidence
- Create research reports and presentations
- Link insights to design opportunities

**Triggers:** "synthesize research", "analyze interviews", "process research"

**Use cases:**
- Analyzing user interview data
- Synthesizing survey responses
- Creating research readouts and reports
- Identifying design opportunities from research

---

### 4. component-documenter
Auto-generate component documentation from Figma designs.

**Features:**
- Extract component properties, variants, and instances from Figma
- Generate usage guidelines and best practices
- Document component anatomy (spacing, colors, typography)
- Create do's and don'ts examples
- Export to Storybook, Notion, Confluence, or Markdown
- Include visual examples and code snippets

**Triggers:** "document component", "generate component docs", "create component guide"

**Use cases:**
- Documenting design system components
- Creating developer handoff documentation
- Building component libraries and style guides
- Maintaining design system documentation

**Integration:** Works with Figma MCP for direct component extraction

---

### 5. design-critique-notes
Document design critiques, feedback sessions, and design reviews.

**Features:**
- Capture feedback and discussion points
- Track action items and decisions made
- Link to specific design iterations and files
- Document attendees and context
- Generate follow-up tasks
- Maintain critique history and evolution

**Triggers:** "log critique", "document feedback", "design review notes"

**Use cases:**
- Recording design critique sessions
- Tracking design feedback and iterations
- Documenting stakeholder reviews
- Maintaining design decision history

**Complements:** design-session-logger (critique-specific vs general session logging)

---

### 6. design-handoff-specs
Generate comprehensive developer handoff documentation and specifications.

**Features:**
- Extract design specs from Figma (spacing, sizing, colors, typography)
- Generate redline/measurement annotations
- Document component states, interactions, and animations
- Create responsive breakpoint specifications
- Export asset requirements and guidelines
- Generate design tokens and style variables
- Include accessibility requirements (ARIA labels, focus states, etc.)
- Create platform-specific specs (iOS, Android, Web)
- Export to developer-friendly formats (JSON, CSS variables, Markdown)

**Triggers:** "create handoff", "generate specs", "developer handoff", "redline specs"

**Use cases:**
- Preparing designs for development handoff
- Creating detailed implementation specifications
- Documenting interaction states and animations
- Generating design tokens for design systems
- Ensuring design-development alignment
- Reducing back-and-forth questions from developers

**Integration:** Works with Figma MCP for direct design extraction and measurement

---
